# Project Progress

本文档用于持续跟踪 JobPilot 的项目进度，作为后续开发和验收的执行清单。

最后更新：2026-03-31

## 状态定义

- 已完成：功能已落地，可实际使用，或已有稳定代码与基础验证
- 半完成：主能力已实现，但仍缺真实场景验证、覆盖扩展或发布收口
- 未完成：尚未开始，或距离可用仍有明显缺口

## 当前结论

项目当前处于“可本地使用的 Chrome 扩展 + 已完成一轮版本发布 + 真实站点兼容性仍待推进”的阶段。

简化判断：

- 产品形态：已成型
- 核心功能：大部分已完成
- 多站点适配：刚进入可扩展阶段
- 真实可用性：仍需要逐站验证

## 已完成

- Chrome MV3 扩展基础结构已完成，包含 `background`、`content scripts`、`popup`、`sidepanel`
- 本地加载使用流程已完成，无需构建，仓库可直接作为 unpacked extension 使用
- 表单检测、规则匹配、自动填写主流程已打通
- AI 字段补全链路已接通
- 多资料模板管理已完成第一版，支持切换、复制、删除、导入、导出
- PDF 简历导入已落地，支持本地解析和 AI 解析
- 简历文件上传已落地，包含自动上传和手动兜底思路
- 填写结果报告、诊断信息、历史记录已落地
- 支持多家 AI 服务商，包含 DeepSeek、Gemini、通义、GLM、Kimi、Claude、Ollama
- 站点适配框架已建立，支持通用适配器和站点特化适配器
- `china-taiping` 站点适配已实现一版，并补充了重复区块自动新增逻辑
- 发布与维护文档已补齐，包含 `README.md`、`CHANGELOG.md`、`PRIVACY.md`、`COMPATIBILITY.md`、`CONTRIBUTING.md`
- 本地 Node 逻辑测试可运行，当前测试结果为 `21/21 passed`
- 资料存储已补上自动快照备份，保存资料时会保留最近 `7` 版历史，降低误覆盖后的恢复成本
- 资料模型已补上“站点覆盖资料层”底座，当前填写链路会按 hostname 合并站点差异字段，避免把平台特有字段硬塞进全局主资料
- 侧边栏设置页已补上 profile 快照列表与一键恢复入口，恢复前会自动再备份当前状态
- 侧边栏资料页已补上“站点覆盖资料”通用入口，支持按当前 hostname 编辑 JSON patch、保存覆盖并预览合并结果

## 半完成

- 站点适配框架已搭好，但真正做深的适配站点还很少，目前主要集中在 `china-taiping`
- `china-taiping` 已完成真实浏览器复测，当前确认“可进入表单页 + 可检测 + 可基础填写 + 可新增重复区块”，但 AI 填写与简历上传尚未复测，所以站点总状态仍保留为 `partial`
- 重复区块自动新增能力已经实现，但需要更多真实页面验证稳定性
- 资料 schema 仍在扩展中，最近仍在新增基础字段并接入侧边栏读写流
- 站点覆盖资料层已接入填写链路，也有了通用 JSON 编辑/预览入口，但还没有结构化字段卡片和细粒度回滚 UI
- AI 补全链路可用，但实际填写质量仍依赖页面字段语义和用户资料完整度
- 本地测试覆盖了核心逻辑，但还缺真实 DOM 流程和真实站点回归验证
- 发布整理已完成一轮，但本地分支目前仍比远端多 1 个提交，仍有未同步改动

## 未完成

- Boss 直聘、牛客校招、智联招聘、51Job、猎聘等中文站点尚未完成系统验证
- Greenhouse、Lever、Workday、SuccessFactors 等海外 ATS 尚未完成系统验证
- 真实站点兼容矩阵尚未持续补全，多数目标站点仍是 `untested`
- 跨域 `iframe` 场景没有通用解决方案，仍受浏览器安全限制
- 某些自定义上传控件仍需要进一步适配，当前可能需要人工介入
- 还没有形成覆盖真实投递流程的系统化回归测试方案
- 还没有形成明确的“支持站点清单”和“已验证版本基线”

- 形成适配新增规范，约束字段识别、重复区块、控件交互、诊断输出

### P2：把验证和维护流程补完整

- 增加针对真实 DOM 的回归测试样例
- 固化 debug 导出文件的使用流程，失败案例统一留档
- 每验证一个站点，就同步更新 `COMPATIBILITY.md` 和 `CHANGELOG.md`
